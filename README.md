# thermal_frame_normalizer

Интерактивный desktop-инструмент для подбора параметров коррекции виньетки и неравномерной яркости на термокадрах.

Что внутри:

- отдельный слой логики обработки и отдельный слой UI;
- переключение между оценкой фона через `GaussianBlur` и `morphologyEx`;
- дополнительные методы оценки фона: `flat-field` по опорному кадру и радиальная модель;
- несколько способов коррекции: multiplicative flat-field, subtractive correction, homomorphic correction и `CLAHE`;
- динамические контролы: при смене метода UI перестраивает набор параметров под актуальный алгоритм;
- сохранение и загрузка пресетов параметров в `JSON`;
- пакетная обработка целой папки текущими настройками;
- предпросмотр `original / background / corrected` и сохранение результата.

Подробный русский справочник по всем методам и настройкам лежит в файле [СПРАВОЧНИК_ПО_МЕТОДАМ.md](<W:\PycharmProjects\_IWT_\thermal_frame_normalizer\СПРАВОЧНИК_ПО_МЕТОДАМ.md>).

```

```

## Возможности UI

- `Open image` загружает одиночный кадр для подстройки;
- `Load preset` и `Save preset` работают с настройками методов и бегунков через `JSON`;
- `Batch folder` прогоняет всю выбранную директорию через текущие параметры и сохраняет результат в указанную папку;
- при смене метода оценки фона или коррекции интерфейс показывает только актуальные параметры.

## Идея обработки

Базовый путь такой:

1. перевести кадр в grayscale;
2. оценить плавный фон, который моделирует виньетку;
3. скорректировать кадр одним из методов;
4. при необходимости слегка растянуть контраст по процентилям.

Такой подход обычно помогает, если затемнение действительно низкочастотное и не меняется хаотично от кадра к кадру.

## Что ещё можно пробовать

- `flat-field calibration`: снять эталонный однородный кадр и использовать его как реальную карту неравномерности;
- радиальную модель виньетки: аппроксимировать затемнение функцией от расстояния до центра;
- `rolling-ball` или морфологическую оценку фона на больших ядрах;
- после коррекции добавить умеренный `CLAHE`, если нужно вернуть локальный контраст;
- работать в `uint16/float`, если важна не только визуализация, но и последующая количественная обработка.
